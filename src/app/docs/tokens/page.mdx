<h1 id="intro" class="docs-title scia-typo-uppercase scia-typo-normal scia-typo-black">Tokens</h1>

<p class="intro">
Tokens are the foundation of Scia CSS.

They are **CSS custom properties** (variables) defined in <code class="scia-code">:root</code>, used by components and utilities to stay consistent and themeable. Instead of hardcoding values (px, colors, shadowsâ€¦), Scia CSS references tokens.

This makes it easy to:
- keep the design consistent
- override the look per project
- evolve the framework without rewriting components

Scia CSS defines token groups for typography, radii, shadows, transitions and spacing.
</p>

<br/>

### Where tokens live

All tokens are defined under <code class="scia-code">:root</code> blocks in <code class="scia-code">scia-tokens.css</code>, using multiple <code class="scia-code">`:root { ... }`</code>.

<br/>
---





<h2 id="how-override" class="scia-h2 docs-subtitle">How to override tokens in your project</h2>

To customize Scia CSS, override tokens **after** importing Scia CSS.

```css
:root {
  --scia-radius-md: 1rem;
  --scia-shadow-sm: 0 1px 2px rgba(0, 0, 0, 0.08);
}
```

<div class="scia-blockquote-warning">
<p class="blockquote-paragraph">Color tokens are usually handled in the **Themes** page (because they have light/dark/auto structure). This page focuses on the shared, non-color system tokens.</p>
</div>

<h2 id="recommended" class="scia-h2 docs-subtitle">Recommended approach</h2>

- Treat tokens as the single source of truth.
- Prefer utilities/components that reference tokens rather than creating one-off values.
- Override tokens per project instead of editing Scia CSS source files.

For color tokens and light/dark/auto theme structure, see the [Themes documentation](/docs/themes).
